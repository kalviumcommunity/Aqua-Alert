NumPy Vectorization Walkthrough

1. The Goal

In this demonstration, we are comparing two approaches to executing element-wise numerical operations using NumPy arrays: a standard Python Loop vs a Vectorized Operation.

Why do we care? Python loops are slow for mathematical operations. Vectorization delegates the calculations to optimized C code, making it drastically faster and more readable.

2. Loop-Based Approach

Here is what a traditional loop in Python looks like when doing element-wise logic:

def loop_based_operation(a, b):
    result = np.zeros_like(a)
    for i in range(len(a)):
        result[i] = a[i] * b[i] + 5
    return result

While correct, this forces Python to individually evaluate each element via the interpreter.

3. Vectorized Approach

Now, let's look at the Vectorized version:

def vectorized_operation(a, b):
    # Clean, concise, and incredibly fast
    return a * b + 5

NumPy inherently understands that a and b are multidimensional arrays. We just express the math directly!

4. Verification

We ran both functions on the same inputs:

Array a: [1 2 3 4 5]
Array b: [10 20 30 40 50]

Loop result:       [15 45 95 165 255]
Vectorized result: [15 45 95 165 255]

✅ Outcomes match exactly.

Vectorization provides the same result, cleaner code, and optimal performance!
